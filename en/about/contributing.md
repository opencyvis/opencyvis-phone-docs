# Contributing

Thanks for your interest in contributing to OpenCyvis!

## Getting Started

1. Fork the repository
2. Clone your fork
3. Create a feature branch
4. Make your changes
5. Run tests
6. Submit a pull request

## Building

```bash
cd android
./gradlew assembleDebug
```

## Running Tests

```bash
cd android
./gradlew testDebugUnitTest
```

## Code Style

- Kotlin: follow standard Kotlin conventions
- Write tests for new features
- Keep commits focused and well-described

## Reporting Issues

Use [GitHub Issues](https://github.com/opencyvis/opencyvis-phone/issues). Include:
- Device model and Android version
- Steps to reproduce
- Expected vs actual behavior
- Logs if available

## Security

If you find a security vulnerability, please report it privately via email rather than opening a public issue.
